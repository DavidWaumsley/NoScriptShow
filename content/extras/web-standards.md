<h1 class="post-title-only">Web Standards</h1>

Modern web standards are the **shared** technical and design rules that keep the web open, interoperable, and robust across devices and browsers. They are defined by bodies like the W3C and WHATWG and cover HTML, CSS, JavaScript, accessibility, performance, and security practices.

In practice, this means we:

- Use well‑structured, semantic HTML to give content clear meaning.
- Separate content, presentation, and behavior (HTML, CSS, JavaScript) so sites are maintainable and resilient.
- Aim for cross‑browser, cross‑device compatibility instead of relying on proprietary or fragile features.

Standards evolve, so “modern” is an ongoing commitment to stay aligned with current best practices, not a one‑time setup.



## Why it matters

Following modern web standards makes the site more reliable, accessible, and future‑friendly. It reduces technical debt, improves performance, and helps ensure the site works well for more people, in more places, for longer.

For this brand, using web standards means:

- Our site is easier to update, extend, and integrate with other tools.
- Visitors benefit from faster load times, better accessibility, and fewer glitches.
- We avoid unnecessary lock‑in to proprietary technologies where an open standard exists.

This approach supports our broader goals of inclusivity, sustainability, and professional quality.



## Core principles we follow

We treat modern web standards as a set of guiding principles, not just a checklist of technologies:

- **Progressive enhancement**: Start with a solid, accessible core that works in basic conditions, then layer on enhancements for capable browsers and devices.
- **Semantic structure**: Use HTML elements for their intended purpose so content is meaningful to both people and machines (browsers, search engines, assistive tech).
- **Responsive design**: Design layouts, typography, and media that adapt gracefully to different screen sizes and orientations.
- **Performance and resilience**: Favour lightweight assets, efficient code, and graceful fallbacks over fragile, heavy solutions.
- **Security and privacy**: Use secure protocols (HTTPS), avoid unsafe patterns, and respect user data.

We still use modern frameworks and tools where appropriate, but we expect them to build on web standards, not replace them.



## How website owners apply this

Website owners don’t need to know every technical detail, but they can still make decisions that reinforce standards‑based practice:

- Prefer solutions that work in a regular browser without special plugins or proprietary viewers.
- Avoid demanding features that rely on “magic” if they create performance or accessibility problems.
- Keep content structured with meaningful headings, lists, and labels rather than styling alone.
- Encourage use of responsive layouts and media that work well on mobile, tablet, and desktop.
- Periodically review the site for outdated technologies (unsupported APIs, insecure embeds, obsolete formats) and plan to replace them.
